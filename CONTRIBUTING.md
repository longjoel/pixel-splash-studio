# Contributing Guide

Thanks for helping improve Pixel Splash Studio! We welcome contributions of all
types: code, design, docs, bug reports, and feature ideas.

## Quick Start

- Fork the repo and create a branch from `main`.
- Keep PRs focused and small when possible.
- Run a local build before submitting:
  - `dotnet build`

## What to Work On

- Check issues for bugs and feature requests.
- If you're proposing a larger change, open an issue first to align on scope.
- UI and tool improvements are always welcome.

## AI-Assisted Contributions

AI-generated or AI-assisted code is welcome. Please:

- Review the output carefully for correctness and style.
- Mention in the PR description what tools you used and how you verified the
  change.
- Ensure generated code follows existing patterns in the repo.

## Code Style and Expectations

- Prefer clear, readable code over cleverness.
- Match existing naming and structure.
- Add concise comments only when the intent is not obvious.

## Pull Request Checklist

- Builds locally (`dotnet build`).
- Describes the problem and the solution.
- Notes any tradeoffs or known limitations.

## Community

Be kind and constructive. If something is unclear, ask questions in the issue
or PR.
